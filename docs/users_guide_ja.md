# Rclone Config App ユーザーガイド

このガイドでは、Rclone Config Appを使用してクラウドストレージ接続を管理する方法を説明します。

## 概要

Rclone Config Appは以下の機能を提供するグラフィカルインターフェースです：
- クラウドストレージ接続の設定
- 既存の接続の管理
- 接続のテスト
- ストレージ内容の表示
- 接続の削除

## はじめに

1. アプリケーションを起動
2. アプリは自動的にシステム上のrcloneを検出
3. rcloneが見つからない場合：
   - デフォルトパスを使用
   - またはrcloneのインストール先パスを入力

## 新しいリモートの追加

1. クラウドプロバイダーボタンをクリック：
   - Google Drive
   - OneDrive
   - Dropbox
   - Box

2. リモートの名前を入力：
   - わかりやすい名前を選択
   - 文字、数字、アンダースコアのみ使用可能
   - 例：「personal_gdrive」や「work_onedrive」

3. 認証プロセスに従う：
   - ブラウザウィンドウが自動的に開く
   - クラウドサービスにログイン
   - rcloneにアクセス許可を与える
   - アプリに戻る

4. 確認を待つ：
   - アプリが接続を確認
   - 完了すると「Remote configured successfully!」と表示
   - 新しいリモートがリストに表示される

## 既存のリモートの管理

### リモートの表示
- 設定済みのリモートがメインウィンドウに表示
- リモートをクリックして選択
- 選択したリモートの詳細が表示

### リモートのテスト
1. リストからリモートを選択
2. 「Check Remote」をクリック
3. アプリが以下を実行：
   - 接続をテスト
   - ストレージ使用量情報を表示
   - 最近のファイル（最大100件）を表示

### リモートの削除
1. 削除したいリモートを選択
2. 「Delete Remote」をクリック
3. リモートが設定から削除される
4. リモートリストが自動的に更新

## インターフェースの理解

### メインウィンドウ
- 上部：ステータスメッセージ付きアプリケーションヘッダー
- 中央：設定済みリモートのリスト
- 下部：新規リモート追加用のクラウドプロバイダーボタン

### リモートリスト
- すべての設定済みリモートを表示
- クリックでリモートを選択
- 選択中のリモートは強調表示
- チェック時にステータスとファイル情報を表示

### ステータスメッセージ
- ヘッダーの下に表示
- 表示内容：
  - 設定の進行状況
  - 接続状態
  - エラーメッセージ
  - 成功の確認

### アクションボタン
- Check Remote：選択した接続をテスト
- Delete Remote：選択したリモートを削除
- クラウドプロバイダーボタン：新規リモート追加

## 一般的なタスク

### Google Driveの設定
1. 「Google Drive」ボタンをクリック
2. リモートの名前を入力
3. Googleアカウントにログイン
4. rcloneに権限を付与
5. 確認を待つ

### OneDriveの設定
1. 「OneDrive」ボタンをクリック
2. リモートの名前を入力
3. Microsoftアカウントにログイン
4. rcloneに権限を付与
5. 確認を待つ

### Dropboxの設定
1. 「Dropbox」ボタンをクリック
2. リモートの名前を入力
3. Dropboxアカウントにログイン
4. rcloneに権限を付与
5. 確認を待つ

### Boxの設定
1. 「Box」ボタンをクリック
2. リモートの名前を入力
3. Boxアカウントにログイン
4. rcloneに権限を付与
5. 確認を待つ

## トラブルシューティング

### 認証の問題
- インターネット接続を確認
- アプリを閉じて再度開く
- クラウドサービスにアクセス可能か確認
- リモートを削除して再作成を試みる

### 接続の問題
- リモートを選択して「Check Remote」をクリック
- インターネット接続を確認
- クラウドサービスが正常に動作しているか確認
- アプリケーションを再起動

### 設定エラー
- rcloneが正しくインストールされているか確認
- 設定のrcloneパスを確認
- 適切な権限があるか確認
- リモートを削除して再追加を試みる

### ブラウザ認証
- アプリがデフォルトブラウザを開く
- アプリケーションからのポップアップを許可
- 認証プロセスを完了
- アクセス許可後にアプリに戻る

## ベストプラクティス

1. リモートの命名
   - わかりやすい名前を使用
   - プロバイダーの種類を含める
   - 目的や場所を追加
   - 例：「personal_gdrive_backup」

2. 定期的なテスト
   - リモートを定期的にテスト
   - ストレージ使用量を確認
   - ファイルアクセスを確認
   - 問題がないかモニター

3. 整理
   - リモートリストを整理
   - 未使用のリモートを削除
   - リモートの目的を文書化
   - 明確な命名規則を維持

## アプリケーションの終了

1. 「Close Application」ボタンをクリック
2. アプリは以下を実行：
   - 一時ファイルのクリーンアップ
   - 設定の保存
   - 適切な終了

## ヘルプの取得

問題が発生した場合：
1. ステータスメッセージを確認
2. rcloneのインストールを確認
3. リモート接続をテスト
4. トラブルシューティングセクションを参照
5. 特定のプロバイダーについてはrcloneのドキュメントを確認
